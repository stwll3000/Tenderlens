## 🚀 Как запустить бота

### 1. Установить зависимости
```bash
pip install python-telegram-bot==20.7
```

### 2. Создать бота через @BotFather

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`
3. Укажите имя бота: `TenderLens Bot`
4. Укажите username: `tenderlens_bot` (или другой доступный)
5. Скопируйте токен бота

### 3. Обновить .env

```env
TELEGRAM_BOT_TOKEN=1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
TELEGRAM_ADMIN_ID=your-telegram-user-id
```

Получить User ID можно через [@userinfobot](https://t.me/userinfobot)

### 4. Запустить бота

```bash
python bot/main.py
```

Или через модуль:
```bash
python -m bot.main
```

### 5. Протестировать команды

Откройте бота в Telegram и попробуйте:
- `/start` — приветствие
- `/stats` — статистика по 6000 лотам
- `/search строительство` — поиск закупок
- `/top_niches` — топ регионов

## 🔧 Настройка Supabase

### Проблема

Текущий connection string не работает:
```
FATAL: (ENOTFOUND) tenant/user postgres.dnpjcxjjavzjmtfzlrip not found
```

### Решение

**Вариант 1: Получить новый connection string** (рекомендуется)

1. Откройте [Supabase Dashboard](https://supabase.com/dashboard)
2. Выберите проект TenderLens (или создайте новый)
3. Перейдите в **Settings → Database**
4. Скопируйте **Connection string** (Transaction mode, порт 6543)
5. Замените `[YOUR-PASSWORD]` на ваш пароль
6. Обновите `.env`:
```env
DATABASE_URL=postgresql://postgres.[PROJECT_REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres
```

**Вариант 2: Работать с JSON** (текущий режим)

Бот и дашборд уже работают с JSON-файлами:
- Данные: `data/lots_multi_regions_6000_20260425.json`
- PostgreSQL можно настроить позже
- Для 6000 лотов JSON достаточно

### Создание таблиц в Supabase

После получения нового connection string:

```bash
# Применить миграции
alembic upgrade head

# Загрузить данные
python db/load_multi_regions.py

# Проверить подключение
python db/check_connection.py
```

Или создайте таблицы вручную через SQL Editor (см. SETUP_SUPABASE.md)
