# TenderLens — План работ

## Обзор проекта

**Цель:** аналитическая платформа рынка госзакупок РФ  
**Стек:** Python · PostgreSQL · Streamlit · Telegram Bot  
**Горизонт:** 20 недель до первого платящего клиента  
**Бюджет на старт:** 0 ₽

---

## ФАЗА 1 — Сбор данных (недели 1–4)

### Неделя 1–2: Первый парсер

- [ ] Изучить структуру XML-фидов zakupki.gov.ru
- [ ] Зарегистрироваться на ЕИС, получить доступ к API
- [ ] Написать `scraper/fetch_lots.py` — базовый парсер лотов
- [ ] Парсить поля: номер лота, заказчик, регион, ОКПД2, НМЦ, цена победы, ИНН победителя, число заявок, даты
- [ ] Сохранять сырые данные в JSON/CSV как бэкап
- [ ] Получить первые **1 000 лотов** и проверить качество

**Чекпоинт:** файл `lots_sample.csv` с 1000 строк, все поля заполнены

### Неделя 3–4: База данных и автосбор

- [ ] Установить PostgreSQL локально (или Railway для облака)
- [ ] Создать схему БД (`db/models.py` через SQLAlchemy)
- [ ] Написать `db/loader.py` — загрузка данных из парсера в БД
- [ ] Настроить дедупликацию по номеру лота
- [ ] Написать GitHub Actions workflow для ежедневного обновления
- [ ] Накопить **30 000+ лотов** по 2–3 регионам

**Чекпоинт:** БД работает, данные обновляются автоматически

## ФАЗА 2 — Аналитика (недели 5–10)

### Неделя 5–7: EDA и метрики

- [ ] `analytics/competition.py` — конкурентность по категориям/регионам
- [ ] `analytics/pricing.py` — расчёт α (снижение цены), распределения
- [ ] `analytics/winners.py` — профиль победителей, win rate по заказчику
- [ ] `analytics/timing.py` — временные паттерны публикаций
- [ ] Оформить выводы как Jupyter Notebook с графиками

**Чекпоинт:** 5 нетривиальных инсайта, оформленных в ноутбуке

### Неделя 8–10: Скоринг и аномалии

- [ ] `analytics/scoring.py` — скоринг привлекательности ниш
- [ ] `analytics/anomalies.py` — флаги подозрительных лотов
- [ ] Тесты `pytest` на функции расчёта метрик
- [ ] Сформировать таблицу топ-20 ниш по скорингу

**Чекпоинт:** автоматический рейтинг ниш обновляется вместе с данными

## ФАЗА 3 — Продукт (недели 11–20)

### Неделя 11–13: Streamlit дашборд

- [ ] `dashboard/app.py` — основной файл
- [ ] Фильтры: регион, категория ОКПД2, заказчик, период

**Чекпоинт:** бот работает и присылает алерты в реальном времени

## Риски и решения

| Риск | Решение |
|---|---|
| Блокировка IP при парсинге | Задержки между запросами, ротация User-Agent |
| Изменение структуры API | Версионировать парсер, мониторить изменения |
| Маленький датасет | Начать с XML-архивов data.gov.ru |
| Нет клиентов | Начать с тематических Telegram-каналов по тендерам |
